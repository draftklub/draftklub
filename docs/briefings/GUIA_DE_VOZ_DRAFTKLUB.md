# DraftKlub — Guia de Voz do Produto

> Sistema de identidade verbal do DraftKlub.
> Este documento define como o produto fala — em UI, marketing,
> emails, suporte, redes sociais.
>
> Ancorado na tagline: **"Onde o Klub acontece."**
>
> Use como referência pra qualquer texto que sai com o nome DraftKlub.

---

## 1. Princípios fundadores

### 1.1 Quem somos verbalmente

DraftKlub fala como **um amigo do clube** — alguém que entende o jogo,
respeita a tradição, mas usa as ferramentas modernas. Não é o sistema
corporativo cinza, nem o app gamificado infantil.

Três pilares:

```
PRESENTE      Falamos no presente do indicativo.
              "Acontece", "joga", "reserva", "vence".
              Não em condicional ou futuro distante.

ATIVO         Sujeito faz a ação.
              "Você reserva" em vez de "Reserva é feita".
              "O Klub administra" em vez de "Administração ocorre".

CONCRETO      Substantivos visíveis, não conceitos abstratos.
              "Quadra", "torneio", "ranking" — palavras que o
              jogador conhece e usa todo dia.
              Evitar: "experiência", "solução", "ecossistema".
```

### 1.2 Anti-mantra (o que NÃO somos)

```
DraftKlub NÃO é:
- Um sistema. É um Klub.
- Um software. É a vida do Klub.
- Uma ferramenta. É onde tudo acontece.
- Uma plataforma. É onde o jogo acontece.
- Uma app. É a sede digital do clube.
```

Use isso em pitches, briefs internos, decisões de copy.

## 2. Vocabulário oficial

### 2.1 Palavra-chave do produto: ACONTECER

Toda a microcopy gira em torno desse verbo. Variações úteis:

```
acontece        |  acontecendo  |  aconteceu
o que acontece  |  acontece aqui |  vai acontecer
faça acontecer  |  está acontecendo
```

Use em headers, empty states, notificações, callouts.

### 2.2 Palavras nossas (preferidas)

| Domínio  | Use                              | Em vez de                            |
| -------- | -------------------------------- | ------------------------------------ |
| Lugar    | Klub, quadra                     | clube, instalações                   |
| Ação     | reservar, jogar, vencer          | agendar, executar, finalizar         |
| Pessoas  | sócio, jogador, comissão         | usuário, membro, cliente             |
| Tempo    | hoje, agora, próximo             | atual, vigente, subsequente          |
| Estado   | confirmado, livre, ocupado       | aprovado, disponível, indisponível   |
| Coletivo | o Klub, a comunidade             | os usuários, a base                  |
| Produto  | DraftKlub, o app, o Klub digital | a plataforma, o sistema, a aplicação |

### 2.3 Palavras proibidas (corporativês)

Lista negra. Se alguém escrever isso, repensa:

```
❌ usuário (use: jogador, sócio, comissão, staff)
❌ plataforma (use: DraftKlub, o app, o Klub)
❌ ecossistema (não use, ponto)
❌ solução (use: o que ela faz, especificamente)
❌ funcionalidade (use: o nome da feature)
❌ experiência (use: o que ela é, não a sensação)
❌ jornada (use: o caminho, ou nada)
❌ engajamento (não use)
❌ stakeholder (use: quem, especificamente)
❌ gestão (use: administra, organiza)
❌ digital transformation (lol)
```

### 2.4 Termos técnicos do domínio (intocáveis)

Estes vêm do tênis, do clube, do esporte. Não traduzir, não suavizar:

```
saque · break · ace · double-fault · winner · forçado · não-forçado
singles · doubles · tie-break · let · deuce · vantagem
chave · cabeça-de-chave · pré-classificatório · semifinal · final
ranking · pontos · rating · ELO
sócio · convidado · comissão esportiva · staff · síndico
```

Quando aparecerem na UI, mantém a forma do esporte. Tradução caseira
afasta o público profissional do clube.

## 3. Microcopy patterns

### 3.1 Headers de página

Curto, presente do indicativo, com ação implícita.

```
✅ Hoje no Klub
✅ Reservas
✅ Torneios em andamento
✅ Sua próxima partida
✅ Ranking — Tennis Singles

❌ Bem-vindo ao seu painel de gestão
❌ Visualização de calendário
❌ Lista de reservas confirmadas
❌ Sistema de torneios e classificações
```

### 3.2 Botões (CTAs)

Verbos no infinitivo, diretos. Sem "Por favor", sem "Clique aqui".

```
✅ Reservar
✅ Estender +30min
✅ Ver detalhes
✅ Cancelar reserva
✅ Confirmar resultado
✅ Adicionar jogador
✅ Entrar
✅ Sair

❌ Clique para reservar
❌ Confirmar a operação de cancelamento
❌ Realizar agendamento
❌ Submeter formulário
```

### 3.3 Empty states

Foco em ação. Cada estado vazio é uma oportunidade pro próximo passo.

```
✅ Nenhuma reserva ainda. [Reservar quadra]
✅ Sem torneios neste mês. [Ver próximos]
✅ Você ainda não está em nenhum ranking. Jogue uma partida pra começar.
✅ Nada acontecendo aqui agora. Volte mais tarde.

❌ Você não possui registros para visualizar.
❌ A lista está vazia.
❌ Nenhum item disponível no momento.
```

### 3.4 Estados de loading

Discreto, presente. Não use spinner sem texto se a ação for longa.

```
✅ Carregando…
✅ Reservando sua quadra…
✅ Aplicando o resultado…
✅ Atualizando o ranking…

❌ Aguarde, processando sua solicitação
❌ Por favor, não feche a janela
```

### 3.5 Mensagens de erro

Seja honesto. Diga o que aconteceu e o que fazer.

```
✅ Quadra ocupada nesse horário. Escolha outro.
✅ Você já tem reserva nesse horário (Quadra 2 • 19:00).
✅ Banda prime não permite convidados.
✅ Não conseguimos conectar. Tenta de novo?

❌ Erro ao processar requisição.
❌ Operação não permitida.
❌ Falha no sistema. Contate o administrador.
❌ HTTP 409 — Conflict
```

### 3.6 Mensagens de sucesso

Curto, com fato + próxima ação se houver.

```
✅ Reservado! Quadra 1 • 19:00–20:00.
✅ Resultado confirmado. Ranking atualizado.
✅ Convidado adicionado.

❌ Sua reserva foi processada com sucesso.
❌ Operação realizada com êxito.
```

### 3.7 Notificações

Personagem ativo + ação concreta.

```
✅ João reservou a Quadra 2 das 19h às 20h.
✅ Pedro venceu sua partida 6-4, 6-3.
✅ Comissão agendou seu jogo: amanhã, 18:30.
✅ Torneio Open de Verão começa em 3 dias.

❌ Nova reserva foi criada no sistema.
❌ Resultado de partida foi atualizado.
❌ Notificação: agendamento.
```

### 3.8 Confirmações destrutivas

Pergunta clara, opções diretas, consequência visível.

```
✅ Cancelar reserva de hoje, 19:00 na Quadra 1?
   Esta ação não pode ser desfeita.
   [Cancelar reserva] [Voltar]

❌ Tem certeza que deseja prosseguir com a operação?
❌ Confirma a exclusão deste registro?
```

## 4. Naming de features

### 4.1 Padrão geral

Nome da feature deve ser **a coisa que ela é**, não "Sistema de X".

```
✅ Calendar (mas chamamos "Hoje no Klub" se for view de hoje)
✅ Torneios
✅ Rankings
✅ Reservas
✅ Membros
✅ Quadras

❌ Sistema de Reservas
❌ Módulo de Torneios
❌ Painel de Membros
❌ Gerenciador de Quadras
```

### 4.2 Aplicação do tema "acontecer"

Não force em todo lugar. Use onde **agrega significado**:

```
SIM (agrega):
- "Hoje no Klub" (em vez de "Calendar")
- "Acontecendo agora" (em vez de "Em andamento")
- "O que aconteceu hoje" (em vez de "Atividade recente")

NÃO FORCE:
- Settings → mantém "Configurações"
- Profile → mantém "Perfil"
- Help → mantém "Ajuda"
```


## 5. Tom por contexto

### 5.1 Para o sócio (B2C)

Caloroso, próximo, esportivo. Trata como amigo do clube.

```
"Sua próxima partida é amanhã."
"João te marcou em uma reserva."
"Você subiu 3 posições no ranking."
```

### 5.2 Para o admin do clube (B2B)

Profissional, eficiente, respeitoso. Trata como gestor sênior.

```
"3 novas reservas hoje."
"Torneio Open de Verão: 24 inscritos."
"Faturamento do mês: R$ 8.450."
```

### 5.3 Para a comissão esportiva

Técnico, preciso, cooperativo. Trata como par especialista.

```
"Resultado pendente: Quartas-final, Quadra 1."
"João vs. Pedro: 6-4, 6-3 — confirmar?"
"Próxima fase libera quando 4 partidas concluírem."
```

### 5.4 Para staff (balcão)

Direto, operacional, rápido. Trata como colega de turno.

```
"Quadra 2 livre agora."
"João Silva quer reservar 19h. Aprovar?"
"Manutenção 14h–16h."
```

### 5.5 Em emails transacionais

Mais explícito, com contexto. Lembrar que email pode ser lido em outra hora.

```
Subject: Sua reserva amanhã • Quadra 1 • 19:00

Olá João,

Você reservou a Quadra 1 amanhã (sex, 26/04) das 19:00 às 20:00.

Quem joga: você + Pedro
Tipo: singles, banda regular

[Ver no app]

Se quiser cancelar, faça até 12h antes pra evitar taxa.

— DraftKlub
Onde o Klub acontece.
```

### 5.6 Em redes sociais

Mais leve, com atitude esportiva. Pode usar emoji esporádico (raqueta, troféu).

```
✅ "Open de Verão começa amanhã. 24 jogadores. Que ganhe o melhor 🎾"
✅ "Hoje no Tennis Carioca: 47 partidas, 12 quadras, 1 ranking que mudou."
❌ "Estamos felizes em anunciar mais um torneio incrível!! 🎉🎉🎉"
```

## 6. Variações da tagline

A tagline principal é: **"Onde o Klub acontece."**

Ela tem variações pra contextos específicos:

```
PRIMÁRIA (logo lock-up, hero, marketing):
"Onde o Klub acontece."

LANDING B2B (vender pra clubes):
"Onde seu Klub acontece todos os dias."

LANDING B2C (atrair sócios):
"O seu Klub acontece aqui."

ASSINATURA EMAIL/RODAPÉ:
"DraftKlub — onde o Klub acontece."

CTA DE LANDING:
"Faça o Klub acontecer."

EVENTOS/PRÓXIMO LANÇAMENTO:
"Algo está acontecendo no Klub."
"Acontece em breve."

PÓS-EVENTO:
"Aconteceu no Klub: [recap]"
```

Não invente novas variações sem necessidade. As acima cobrem 99% dos casos.

## 8. Checklist antes de publicar qualquer texto

Antes de ir ao ar (UI, email, post, anúncio), passa pelo checklist:

```
[ ] Está no presente do indicativo?
[ ] Sujeito faz a ação (não passiva impessoal)?
[ ] Usa palavras concretas (não abstratas)?
[ ] Evita as palavras proibidas (seção 2.3)?
[ ] Usa "Klub" com K quando se refere ao produto?
[ ] CTA é verbo no infinitivo direto?
[ ] Mensagem de erro diz o que fazer agora?
[ ] Length apropriada (curto sempre que possível)?
[ ] O sócio do clube de 60 anos entende?
[ ] Combina com "Onde o Klub acontece"?
```

Se 8 ou mais respondidos sim, está bom.

## 9. Exemplos antes-e-depois (treinamento)

### Caso 1 — Notificação push

```
ANTES:
"Você possui uma nova reserva pendente de aprovação no sistema."

DEPOIS:
"João pediu pra entrar na sua reserva de amanhã."
```

### Caso 2 — Email transacional

```
ANTES:
"Prezado usuário, informamos que sua solicitação de reserva foi
processada com sucesso pelo nosso sistema."

DEPOIS:
"Reserva confirmada: Quadra 1, amanhã 19:00."
```

### Caso 3 — Erro de validação

```
ANTES:
"Operação não permitida. Verifique as condições do clube."

DEPOIS:
"Banda prime não permite convidados. Tente um horário antes das 17h."
```

### Caso 4 — Header de página

```
ANTES:
"Painel de Visualização de Calendário e Reservas"

DEPOIS:
"Hoje no Klub"
```

### Caso 5 — Empty state

```
ANTES:
"Não foram encontrados resultados para os filtros aplicados."

DEPOIS:
"Nenhuma partida desse jogador ainda. Marque a primeira."
```

## 10. Manutenção deste guia

Este guia evolui com o produto. Quando criarem features novas:

1. Adicionar a feature à lista canônica (seção 4.3)
2. Decidir o nome em UI antes de qualquer copy
3. Criar exemplos de microcopy padrões da feature
4. Atualizar este doc

Reviews trimestrais com time de produto + design + marketing.
